## 7. GPIO.HIGH / GPIO.LOW Usage with Context

**Pattern**: `GPIO\.(HIGH|LOW)` with 2 lines of context
**Scope**: `*.py` files

### Active relay logic summary

**LOW = relay ON (active), HIGH = relay OFF (inactive)** for all relay channels.

#### Flash control pattern (most common, repeated across ~20 scripts)

```python
def flashOn():
    GPIO.output(Relay_Ch3, GPIO.LOW)   # Buck converter ON
    GPIO.output(Relay_Ch2, GPIO.LOW)   # Flash lights ON
    print("Flash On\n")

def flashOff():
    GPIO.output(Relay_Ch2, GPIO.HIGH)  # Flash lights OFF
    print("Flash Off\n")
```

#### Attract control pattern (Flash_On.py / Flash_Off.py / Attract_On.py / Attract_Off.py)

```python
def AttractOn():
    GPIO.output(Relay_Ch3, GPIO.LOW)   # Buck converter ON
    if onlyflash:
        GPIO.output(Relay_Ch2, GPIO.LOW)   # Flash also ON
    else:
        GPIO.output(Relay_Ch2, GPIO.HIGH)  # Flash stays OFF
    GPIO.output(Relay_Ch1, GPIO.LOW)   # UV/attract ON

def AttractOff():
    GPIO.output(Relay_Ch1, GPIO.HIGH)  # UV/attract OFF
    GPIO.output(Relay_Ch2, GPIO.HIGH)  # Flash OFF (both paths)
    GPIO.output(Relay_Ch3, GPIO.HIGH)  # Buck converter OFF
```

#### 5.x/Attract_On.py (INVERTED logic -- potential anomaly)

```python
# 5.x/Attract_On.py:52-62
def AttractOff():
    GPIO.output(Relay_Ch3, GPIO.HIGH)  # All HIGH (off)
    GPIO.output(Relay_Ch2, GPIO.HIGH)
    GPIO.output(Relay_Ch1, GPIO.HIGH)

def AttractOn():
    GPIO.output(Relay_Ch3, GPIO.LOW)   # All LOW (on)
    GPIO.output(Relay_Ch2, GPIO.LOW)
    GPIO.output(Relay_Ch1, GPIO.LOW)
```

Note: This `5.x/Attract_On.py` turns ALL three relays on/off, while the `scripts/` versions use the `onlyflash` conditional.

## Summary of Key Findings

1. **BCM/BOARD mode mismatch**: `4.x/FlashOn.py` and `5.x/FlashOn.py` use `GPIO.BOARD` but consume pin numbers from `get_gpio_pins()` which returns BCM values.

2. **Pin aliasing in 5.x/Flash_Off.py**: `Relay_Ch1 = pins["Relay_Ch2"]` -- local variable name does not match the actual channel being controlled.

3. **Hardcoded switch pins**: `off_pin = 16` and `debug_pin = 12` are hardcoded in 8 files across both firmware versions, not managed by `mothbox_paths.py`.

4. **No lgpio/gpiod usage**: The entire codebase uses `RPi.GPIO` exclusively; no `lgpio.gpio_write()` or `gpiod` calls exist.

5. **OldScripts bypass configuration**: Scripts in `OldScripts/` directories hardcode `26/20/21` pin values, completely bypassing `get_gpio_pins()`.

6. **Attract_On.py logic differs**: The top-level `5.x/Attract_On.py` turns ALL three relays on/off unconditionally, while the `scripts/` versions use an `onlyflash` conditional for `Relay_Ch2`.

7. **Module-level pin loading**: `webui/backend/routes/gpio.py:15` calls `get_gpio_pins()` at import time, then also re-calls it inside each handler -- the module-level call is stale if config changes at runtime.
